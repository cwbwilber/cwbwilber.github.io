Java反射

反射机制

作用：通过Java语言中的反射机制可以操作字节码文件。

通过反射机制可以操作代码片段（class文件）

反射机制重要的类：

  • java.lang.Class：代表整个字节码，代表一个类型
  • java.lang.reflect.Method：代表字节码中的方法字节码，代表类中的方法
  • java.lang.reflect.Constructor：代表字节码中的构造方法字节码，代表类中的构造方法
  • java.lang.reflect.Field：代表字节码中的属性字节码，代表类中的成员变量（静态变量）

注解

  • 注解又叫做注释，英文单词是：Annotation

  • 注释Annotation是一种引用数据类型。编译后也生成class文件

  • [修饰符] @interface 注解类型名 {}

  • 注解使用

    • 语法格式：@注解类型名
    • 注解可以出现在类上、属性上、方法上、变量、注解上

内置注解

Deprecated

用@Deprecated注释的元素，不鼓励程序员使用这样的元素，通常是因为它很危险或存在更好的选则

Override

用@Override注释的元素表示一个方法声明打算重写超类的另一个方法

SuppressWarnings

@SuppressWarnings指示应该在该注释元素（以及包含在该注释元素中的所有程序元素）中取消显示指定的编译器警告

元注解

用来标注“注解类型”的“注解”，称为元注解

常见的元注解有

  • Target
  • Retention

Target

Target用来标注“被标注的注解”可以出现在哪些位置上

  • @Target(ElementType.METHOD)：表示只能出现在方法上

Retention

Retention用来标注“被标注的注解”最终保存在哪里

  • @Retention(RetentionPolicy.SOURCE)：保留在Java源文件中
  • @Retention(RetentionPolicy.CLASS)：保留在class文件中
  • @Retention(RetentionPolicy.RUNTIME)：保留在class文件中，并且可以被反射机制所读取
